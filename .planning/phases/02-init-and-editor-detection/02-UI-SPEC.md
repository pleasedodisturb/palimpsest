# Phase 2 — UI Design Contract

> Visual and interaction contract for terminal UI (Rich + Typer). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Note:** This is a CLI tool, not a web frontend. "UI" means terminal output rendered by Rich and interactive prompts via Typer/Rich. The design contract specifies terminal styling, colors, layout patterns, and copywriting for all user-facing output in Phase 2.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Rich + Typer (terminal UI) |
| Preset | PAL_THEME (defined in `palimpsest/console.py`) |
| Component library | Rich widgets (Panel, Table, Tree, Prompt, Confirm, Progress) |
| Icon library | Unicode symbols (not emoji) |
| Font | Terminal default (monospace, user-controlled) |

### Existing Theme Tokens (from Phase 1)

```python
PAL_THEME = Theme({
    "info": "cyan",
    "success": "bold green",
    "warning": "bold yellow",
    "error": "bold red",
    "hint": "dim green",
    "command": "bold magenta",
})
```

Add these to `PAL_THEME` in `console.py`:

## Spacing Scale

Terminal spacing uses blank lines and indentation, not pixel values. All spacing is in character units.

| Token | Value | Usage |
|-------|-------|-------|
| inline | 1 space | Between inline elements (icon + text) |
| indent | 2 spaces | Nested list items, sub-sections |
| paragraph | 1 blank line | Between logical sections in output |
| section | 2 blank lines | Between major phases of init flow |

Exceptions: Rich Panel and Table widgets handle their own internal padding (1 char horizontal, 0 vertical by default). Do not add extra blank lines before/after Panels.

---

## Typography

Terminal text has no font-size control. Typography contract covers weight (bold/dim/normal) and semantic markup.

| Role | Rich Style | Usage |
|------|-----------|-------|
| Body | (no style) | Default explanatory text |
| Label | `[label]` / `[bold cyan]` | Prompt labels, field names, category headers |
| Heading | `[heading]` / `[bold white]` | Section titles within init flow |
| Emphasis | `[bold]` | Key values (project name, editor names, file counts) |
| De-emphasis | `[dim]` | Paths, secondary info, skip reasons |
| Command | `[command]` / `[bold magenta]` | CLI commands referenced in output (`pal sync-config`, `pal check`) |

---

## Color

Terminal colors from PAL_THEME, mapped to semantic roles.

| Role | Style | Usage |
|------|-------|-------|
| Dominant surface | Terminal default (no override) | All body text, backgrounds |
| Informational | `[info]` / cyan | Status messages, counts, progress |
| Success | `[success]` / bold green | Completion messages, file created confirmations |
| Warning | `[warning]` / bold yellow | Conflicts detected, files modified by user |
| Error | `[error]` / bold red | Failures, missing requirements |
| Hint | `[hint]` / dim green | Suggested next steps, tips |
| Command reference | `[command]` / bold magenta | CLI commands in output text |

### Action Colors (dry-run table, scaffold summary)

| Action | Color | Symbol |
|--------|-------|--------|
| Create | green | + |
| Overwrite | yellow | ~ |
| Skip (unchanged) | dim | - |
| Backup | blue | B |

Accent color (magenta/`[command]`) reserved for: CLI command references only. Never used for status messages, counts, or general emphasis.

## Terminal Component Contracts

### 1. Welcome Banner

Shown at start of `pal init`. One Rich Panel.

```
+--------------------------------------------------+
|                                                    |
|  Initializing palimpsest                          |
|  AI-augmented Technical Program Management        |
|                                                    |
+--------------------------------------------------+
```

- Widget: `Panel(title="pal init", border_style="cyan")`
- Content: 2 lines. Tool name bold, tagline normal weight.
- Shown: Always, unless `--quiet` flag is set.

### 2. Interactive Prompts

All prompts use Rich `Prompt.ask()` or `typer.confirm()`.

| Prompt | Type | Default | Choices |
|--------|------|---------|---------|
| Program name | `Prompt.ask` | Current directory name | free text |
| Privacy profile | `Prompt.ask` with choices | "team" | solo, team, open |
| Editor confirmation | `typer.confirm` per editor | True for detected, False for undetected | y/n |
| Conflict resolution | `Prompt.ask` with choices | "skip" | skip, overwrite, backup |
| Git init | `typer.confirm` | True | y/n |

Format for all prompts:
- Label uses `[label]` style (bold cyan)
- Default value shown in brackets: `[default]`
- Invalid input re-prompts (handled by Rich Prompt automatically)

### 3. Editor Detection Summary

Rich Table showing detection results.

```
  Editor Detection
+---------------+----------+----------+
| Editor        | Detected | Selected |
+---------------+----------+----------+
| Cursor        | Yes      | Yes      |
| Claude Code   | Yes      | Yes      |
| Cline         | No       | --       |
+---------------+----------+----------+
```

- Widget: `Table(title="Editor Detection", show_lines=True)`
- "Detected" column: `[editor.detected]Yes[/]` or `[dim]No[/]`
- "Selected" column: `[success]Yes[/]`, `[dim]No[/]`, or `[dim]--[/]` (not offered)
- After table: for each undetected editor, prompt to install anyway

### 4. Privacy Explanation Panel

Shown before privacy profile selection. Rich Panel with structured content.

```
+-- Privacy Model --------------------------------+
|                                                   |
|  palimpsest organizes your project context into   |
|  two categories:                                  |
|                                                   |
|  Public (committed to git):                       |
|    Team meeting notes, daily digests, shared docs |
|                                                   |
|  Private (gitignored):                            |
|    1:1 notes, sensitive context, personal notes   |
|    Critical for AI context but never exposed      |
|                                                   |
+--------------------------------------------------+
```

- Widget: `Panel(title="Privacy Model", border_style="cyan")`
- "Public" label: `[privacy.public]`
- "Private" label: `[privacy.private]`
- Shown: Always (this is a differentiator per CONTEXT.md)

### 5. Dry-Run Preview Table

Shown when `--dry-run` is passed. Rich Table listing all planned operations.

```
  Dry Run Preview
+----------+----------------------------------------------+----------+
| Action   | Path                                         | Reason   |
+----------+----------------------------------------------+----------+
| + create | docs/PROGRAM_OVERVIEW.md                     |          |
| + create | docs/palimpsest/01-problem-statement.md      |          |
| ~ overwrite | CLAUDE.md                                 | modified |
| - skip   | .palimpsest/manifest.json                    | exists   |
+----------+----------------------------------------------+----------+

Total: 47 files would be created, 1 overwritten, 1 skipped
```

- Widget: `Table(title="Dry Run Preview", show_lines=True)`
- Action column: styled with action colors (green/yellow/dim)
- Path column: `[path]` style (dim white)
- Footer line: `[info]Total: {N} files would be created, {N} overwritten, {N} skipped[/]`
- Exit after display (no writes)

### 6. Scaffold Progress

Shown during actual file operations. Rich progress display.

- Widget: `console.status("Scaffolding project...")` for the overall operation
- Per-category: print `[info]  {category}[/info] [dim]{count} files[/dim]` as each group completes
- Categories in order: methodology, templates, playbooks, showcase, case study, program docs, memory bank, agent configs, editor configs, gitignore, manifest

### 7. Completion Summary

Shown after successful scaffold. Rich Panel with Tree.

```
+-- Project initialized ---------------------------+
|                                                   |
|  MyProgram                                        |
|                                                   |
|  docs/                        12 files            |
|  docs/palimpsest/             28 files            |
|  .palimpsest/memory-bank/      7 files            |
|  Editor configs:              Cursor, Claude Code |
|  Privacy profile:             team                |
|  Manifest:                    .palimpsest/manifest.json |
|                                                   |
|  Next steps:                                      |
|    pal check     Validate your setup              |
|    pal docs      Browse methodology guides        |
|                                                   |
+--------------------------------------------------+
```

- Widget: `Panel(title="[success]Project initialized[/success]", border_style="green")`
- File counts: `[bold]{count}[/bold]` files
- Editor list: comma-separated, `[bold]`
- Next steps: `[command]` style for commands, normal for descriptions

### 8. Error Panel (inherited from Phase 1)

Already implemented in `errors.py`. No changes needed.

```
+-- error -----------------------------------------+
|                                                   |
|  {error message}                                  |
|                                                   |
|  Hint: {suggested fix}                            |
|                                                   |
+--------------------------------------------------+
```

- Widget: `Panel(title="error", border_style="red")` on stderr
- Existing implementation is sufficient

### 9. Conflict Resolution Display

Shown when re-running init and modified files are detected.

```
File exists: docs/PROGRAM_OVERVIEW.md
  Status: modified by user (checksum mismatch)
  [skip / overwrite / backup] (skip):
```

- First line: `[warning]File exists:[/warning] [path]{path}[/path]`
- Status line: indented 2 spaces, `[dim]`
- Prompt: Rich `Prompt.ask` with choices

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `pal init` (command itself is the CTA) |
| Welcome line 1 | `Initializing palimpsest` |
| Welcome line 2 | `AI-augmented Technical Program Management` |

| Error: not a directory | `Not a valid directory: {path}` + Hint: `Run pal init in a project directory or pass a project name to create one.` |
| Error: permission denied | `Cannot write to {path}: permission denied` + Hint: `Check directory permissions or run from a writable location.` |
| Error: manifest corrupted | `Manifest file is corrupted: {path}` + Hint: `Delete .palimpsest/manifest.json and re-run pal init.` |
| Dry-run footer | `{N} files would be affected. No changes were made.` |
| Scaffold complete | `Project initialized` (panel title) |
| Next step 1 | `pal check` + `Validate your setup` |
| Next step 2 | `pal docs` + `Browse methodology guides` |
| Privacy intro | `palimpsest organizes your project context into two categories:` |
| Privacy public label | `Public (committed to git)` |
| Privacy public desc | `Team meeting notes, daily digests, shared docs` |
| Privacy private label | `Private (gitignored)` |
| Privacy private desc | `1:1 notes, sensitive context, personal notes. Critical for AI context but never exposed.` |
| Profile: solo | `Everything private -- maximally safe for personal use` |
| Profile: team | `Structured public/private split -- recommended for team projects` |
| Profile: open | `Most things public -- for open programs with minimal private data` |
| Conflict prompt | `File exists: {path}` |
| Git init prompt | `No git repository detected. Initialize one?` |
| Editor not detected | `{Editor} not detected. Install config anyway?` |
| Overwrite warning | `This file has been modified since last init.` |
| Backup notice | `Backed up to {path}.bak` |

### Destructive Actions

| Action | Confirmation Approach |
|--------|----------------------|
| Overwrite user-modified file | Per-file `Prompt.ask` with choices: skip (default), overwrite, backup |
| Overwrite existing CLAUDE.md | Same conflict prompt, but default to "backup" (higher risk) |
| Git init in existing directory | `typer.confirm` with default True |

No bulk-destructive operations exist in this phase. All overwrites are per-file with user confirmation.

## Interaction Flow

The complete init flow, in order:

```
1. Welcome banner (Panel)
2. Program name prompt
3. Editor detection (Table + per-editor confirm prompts)
4. Privacy explanation (Panel)
5. Privacy profile selection (Prompt with choices)
6. [if --dry-run] Dry-run preview (Table) -> exit
7. Scaffold progress (status + per-category lines)
8. Completion summary (Panel with next steps)
```

Interruption points:
- Ctrl+C at any prompt: clean exit, no partial writes (nothing written until step 7)
- Conflict resolution: inserted between steps 6 and 7 when re-running init

Quiet mode (`--quiet`):
- Suppresses welcome banner, privacy panel, and scaffold progress
- Still shows prompts (interactive input required)
- Still shows completion summary (reduced to one line: `Initialized {name} ({N} files)`)

Verbose mode (`--verbose`):
- Shows file-by-file output during scaffold (each file path as it's copied)
- Shows full manifest path and checksum count
