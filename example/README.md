# Example: Apex API Platform Migration

This is a **fictional but realistic** example of what a Palimpsest project looks like when it's actually being used. Everything here is made up — the company, the people, the decisions — but the patterns are real. This is what a mid-flight technical program looks like when someone bothers to write things down.

## The Scenario

**Apex Corp** is migrating from a monolithic REST API to a federated GraphQL platform. The program is 4 months in, originally scoped for 6 months, and currently at risk because the auth team is behind and the data team just discovered their schema migration is more complex than estimated.

Sound familiar? Yeah.

## What to Notice

1. **The overview isn't perfect** — it has gaps, some sections are more detailed than others. That's realistic. Nobody maintains perfect docs under pressure.

2. **The dashboard tells the real story** — you can see where the program actually is vs. where the plan says it should be. The gap between those two things is what TPMs manage.

3. **The backlog has parked items** — things that got deprioritized because reality happened. Every program has these. The ones that don't write them down just forget they existed.

4. **The daily digests show context accumulation** — each one is small, but together they paint a picture no single status meeting could capture.

5. **The decisions have rationale** — not just "we decided X" but "we decided X because Y, and here's what we considered." This is what saves you when someone asks "why did we do it this way?" six months later.

## Try It

Open this directory in your AI editor and ask:

```
What's the current status of the Apex migration? What should I be worried about?
```

Then try:

```
Draft a stakeholder update for the VP of Engineering. Focus on timeline risk and the auth blocker.
```

The AI will read these documents and give you answers grounded in actual program state — not hallucinated summaries.
