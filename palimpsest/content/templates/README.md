# Templates

Reusable document templates for running technical programs.

## What These Are

These are starting points, not finished documents. Each template encodes a structure that's been tested across real programs — the sections exist because someone needed them, not because a framework said they should.

Copy the template, replace the placeholders (`[YOUR-PROJECT]`, `[OWNER]`, `[DATE]`), delete sections that don't apply. A shorter document that gets read beats a comprehensive one that doesn't.

## How to Customize

1. **Copy, don't modify originals** — keep templates clean for reuse
2. **Delete irrelevant sections** — no one reads padding
3. **Replace all placeholders** — search for `[` to find them all
4. **Add program-specific sections** — templates are floors, not ceilings
5. **Link related docs** — BRDs link to PRDs, PRDs link to tickets

## Placeholders Used

- `[YOUR-PROJECT]` — program or project name
- `[OWNER]` — responsible person or team
- `[DATE]` — relevant date (created, updated, target)
- `[STAKEHOLDER]` — person or group with interest in the outcome
- `[METRIC]` — measurable indicator of success
