# Playbooks

Practical guides for recurring situations in technical program management.

## What These Are

Playbooks are opinionated how-to guides. They're not templates you fill in — they're approaches you adapt. Each one addresses a specific challenge that comes up repeatedly in cross-functional programs.

## Available Playbooks

| Playbook | What It Covers |
|----------|---------------|
| [Cross-Timezone Collaboration](cross-timezone-collaboration.md) | Working effectively across time zones without burning out on meetings |
| [Decision Documentation](decision-documentation.md) | Capturing decisions so future-you doesn't have to re-litigate them |
| [Reducing Status Meetings](reducing-status-meetings.md) | Replacing information-exchange meetings with async reporting |
| [Remote Onboarding](remote-onboarding.md) | Structured approach to onboarding into a distributed program |

## How to Use

1. Read the playbook that matches your situation
2. Adapt the approach to your team's context
3. Start with the simplest version that works
4. Iterate based on what you learn

These playbooks assume async-first, documentation-heavy programs. If your team is fully co-located and prefers verbal communication, some of this will need adjustment.
